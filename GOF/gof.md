### 一、创建型模式（Creational Patterns）

| 模式                | 说明                                                 | 实际应用案例                                                 |
| ------------------- | ---------------------------------------------------- | ------------------------------------------------------------ |
| **1. 单例模式**     | 保证一个类只有一个实例，并提供全局访问点。           | **日志记录器（Log4j）**、**Spring IOC容器**、**Windows任务管理器** |
| **2. 工厂方法模式** | 定义一个创建对象的接口，但由子类决定实例化哪一个类。 | **Java JDBC的Driver接口**，连接由不同数据库厂商提供          |
| **3. 抽象工厂模式** | 提供一个创建一系列相关或相互依赖对象的接口。         | **Java GUI工具包（如AWT、Swing）**：提供Button、Text等组件系列 |
| **4. 建造者模式**   | 将一个复杂对象的构建与它的表示分离。                 | **StringBuilder**、**构建复杂HTML/XML文档**                  |
| **5. 原型模式**     | 通过复制已有对象来创建新对象。                       | **游戏中的角色/地图克隆功能**，如《星际争霸》的兵种复制      |



------

### 二、结构型模式（Structural Patterns）

| 模式              | 说明                                              | 实际应用案例                                                 |
| ----------------- | ------------------------------------------------- | ------------------------------------------------------------ |
| **6. 适配器模式** | 将一个类的接口转换成客户期望的另一个接口。        | **Java I/O中的InputStreamReader**，**USB转接头软件**         |
| **7. 桥接模式**   | 将抽象部分与实现部分分离，使它们可以独立变化。    | **JDBC桥接ODBC驱动**、**图形库抽象（如绘图API）**            |
| **8. 装饰者模式** | 动态地给对象添加额外职责。                        | **Java I/O的BufferedInputStream、DataInputStream**           |
| **9. 组合模式**   | 将对象组合成树形结构以表示“整体-部分”的层次结构。 | **文件系统目录结构**、**HTML DOM树结构**                     |
| **10. 外观模式**  | 提供一个统一的接口，用来访问子系统中的一群接口。  | **Spring的JdbcTemplate封装数据库访问**、**Windows控制面板**  |
| **11. 享元模式**  | 共享大量细粒度对象，减少内存使用。                | **Java中的Integer.valueOf()缓存**、**游戏中重复场景或粒子对象** |
| **12. 代理模式**  | 为其他对象提供一个代理以控制对它的访问。          | **远程代理（RMI）**、**权限代理（Spring AOP）**、**图像延迟加载** |



------

### 三、行为型模式（Behavioral Patterns）

| 模式                 | 说明                                               | 实际应用案例                                                 |
| -------------------- | -------------------------------------------------- | ------------------------------------------------------------ |
| **13. 责任链模式**   | 将多个处理器链接成一条链，依次处理请求。           | **Servlet过滤器链**、**日志处理框架（Logback）**             |
| **14. 命令模式**     | 将请求封装为对象，支持请求排队、撤销等操作。       | **GUI按钮触发事件**、**事务型系统的操作撤销/恢复功能**       |
| **15. 解释器模式**   | 为语言定义语法规则，并用解释器解释。               | **正则表达式解析器**、**SQL解析器**                          |
| **16. 迭代器模式**   | 提供一种访问容器中元素的方式，而不暴露底层结构。   | **Java中的Iterator接口**、**Python的for...in结构**           |
| **17. 中介者模式**   | 用一个中介对象封装多个对象之间的交互。             | **聊天室系统中的服务器**、**MVC中的Controller**              |
| **18. 备忘录模式**   | 在不破坏封装性的前提下保存对象状态，以便后续恢复。 | **IDE中的撤销/恢复功能（Ctrl+Z）**                           |
| **19. 观察者模式**   | 定义对象间的一对多依赖关系，通知所有观察者。       | **GUI事件监听器（Java Swing）**、**微信公众号订阅推送**      |
| **20. 状态模式**     | 允许对象在其内部状态改变时改变其行为。             | **线程状态切换**、**TCP连接状态机（LISTEN, SYN_SENT, ESTABLISHED等）** |
| **21. 策略模式**     | 定义一系列算法，封装起来使它们可以互换。           | **排序策略（快排/冒泡）**、**Spring中事务策略配置**          |
| **22. 模板方法模式** | 定义一个算法框架，将具体步骤延迟到子类。           | **JDBC中使用模板方法执行SQL**、**钩子函数机制**              |
| **23. 访问者模式**   | 分离算法与对象结构，让算法独立变化。               | **编译器语法树遍历**、**XML解析器（如DOM访问）**             |
