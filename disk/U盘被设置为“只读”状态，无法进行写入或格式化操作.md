好的，格式化U盘时遇到“写保护”是一个常见问题。这通常意味着U盘被设置为“只读”状态，无法进行写入或格式化操作。

方法四：使用Diskpart命令清除只读属性（强力推荐）

这是解决逻辑层面写保护最有效的方法。**请务必谨慎操作，不要选错磁盘！**

1. 以**管理员身份**打开命令提示符（同上）。

2. 依次输入以下命令，每输完一行按一次回车：

   cmd

   ```
   diskpart
   ```

   

   启动磁盘分区工具。

   cmd

   ```
   list disk
   ```

   

   列出电脑上所有的磁盘。根据**大小**确定哪一个是你的U盘（例如，Disk 2，容量为14GB）。

   cmd

   ```
   select disk 2
   ```

   

   选中你的U盘（请将 `2` 替换为你U盘对应的磁盘编号）。

   cmd

   ```
   attributes disk clear readonly
   ```

   

   **这个命令是关键！** 它会清除磁盘的只读属性。

   cmd

   ```
   clean
   ```

   

   **警告：此命令会删除U盘上的所有分区和数据！** 它会将U盘恢复为未分配状态。

   cmd

   ```
   create partition primary
   ```

   

   创建一个新的主分区。

   cmd

   ```
   format fs=ntfs quick
   ```

   

   快速格式化该分区为NTFS格式（你也可以将 `ntfs` 替换为 `fat32` 或 `exfat`）。

   cmd

   ```
   exit
   ```

   

   退出diskpart。

完成以上步骤后，你的U盘应该已经被成功格式化，并且写保护问题已解决。
