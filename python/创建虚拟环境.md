在 Python 中，你可以使用 `venv` 模块在文件夹内创建虚拟环境。以下是详细步骤：

### 1. 打开终端或命令行

- **Windows**：按 `Win + R`，输入 `cmd`，回车。
- **Mac/Linux**：打开终端（Terminal）。

### 2. 进入目标文件夹

使用 `cd` 命令切换到你想创建虚拟环境的目录：

bash

```
cd /path/to/your/folder
```

例如：

bash

```
cd C:\Projects\my_project  # Windows
cd ~/Projects/my_project  # Mac/Linux
```

### 3. 创建虚拟环境

运行以下命令（以 Python 3 为例）：

bash

```
python -m venv venv_name
```

- `venv_name` 是你想给虚拟环境起的名字（通常用 `venv` 或 `.venv`）。

- 例如：

  bash

  ```
  python -m venv venv
  ```

### 4. 激活虚拟环境

- **Windows**：

  bash

  ```
  .\venv_name\Scripts\activate
  ```

  例如：

  bash

  ```
  .\venv\Scripts\activate
  ```

- **Mac/Linux**：

  bash

  ```
  source venv_name/bin/activate
  ```

  例如：

  bash

  ```
  source venv/bin/activate
  ```

激活后，终端提示符前会显示虚拟环境名称（如 `(venv)`）。

### 5. 退出虚拟环境

完成后，输入以下命令退出：

bash

```
deactivate
```

------

### 可选：使用 `virtualenv`（第三方工具）

如果你更喜欢 `virtualenv`，可以这样操作：

1. 安装 `virtualenv`：

   bash

   ```
   pip install virtualenv
   ```

2. 创建虚拟环境：

   bash

   ```
   virtualenv venv_name
   ```

3. 激活/退出方式与 `venv` 相同。

------

### 注意事项

1. 确保你的 Python 版本是 3.3+（`venv` 是标准库）。
2. 虚拟环境文件夹（如 `venv`）通常建议添加到 `.gitignore` 中。
3. 如果遇到权限问题，在 Mac/Linux 前加 `sudo`，或使用 `--without-pip` 参数后手动安装 pip。

### 常用虚拟环境管理命令**

| 命令                                        | 作用                           |
| :------------------------------------------ | :----------------------------- |
| `python -m venv venv_name`                  | 创建虚拟环境                   |
| `source venv_name/bin/activate` (Mac/Linux) | 激活虚拟环境                   |
| `.\venv_name\Scripts\activate` (Windows)    | 激活虚拟环境                   |
| `deactivate`                                | 退出虚拟环境                   |
| `pip list`                                  | 查看已安装的库                 |
| `pip freeze > requirements.txt`             | 导出依赖到 `requirements.txt`  |
| `pip install -r requirements.txt`           | 从 `requirements.txt` 安装依赖 |
